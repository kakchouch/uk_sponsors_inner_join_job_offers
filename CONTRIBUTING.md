# Contributing

Thanks for your interest in contributing.

## Development and Review Policy

This project is developed with AI assistance.
All proposed changes are reviewed by the maintainer before they are merged.

## How to Contribute

1. Open an issue to describe the bug, improvement, or feature idea.
2. Keep changes small, safe, and focused.
3. Follow project coding principles, including PEP8 compliance and DRY.
4. Update relevant documentation when behavior or workflow changes.
5. Open a pull request with a clear summary of what changed and why.

## Pull Request Expectations

- Explain the problem being solved.
- Describe the implementation approach.
- Highlight any trade-offs or known limitations.
- Ensure changes are understandable for beginner developers.

## Maintainer Review

The maintainer performs the final review and decides whether a change is accepted.
